Q) What are drawbacks of propotitional logic? How it can be resolved by predicate logic? Example.

-> The drawbacks of propotitional logic are:

  • We cannot represent relations like ALL, some, or none with propositional logic.
    Example:
    All the girls are intelligent.
    Some apples are sweet.
  • Propositional logic has limited expressive power.
  • In propositional logic, we cannot describe statements in terms of their properties or logical relationships.


   We can solve the propotional logic using predicate logic by following steps:
  • First we convert the given propotional statement to predicate statement.(i.e using variables).
    For example: 2 is even(propositional) to x is even / even(x) (predicate)
  • Use the connectives (∧-AND, ∨-OR, ¬-NOT)
  • Use quantifiers for quantitive statements. (∀-For all, ∃-For some)
    example: All the books are red - ∀x book(x) → red(x)

What do you mean by Skolemnization? Explain the CNF Conversion steps in predicate logic. Also convert "Anything anyone eats and is not killed by is Food" to FOL and equivalent CNF.

-> Skolemnization is the process/step in CNF conversion from FOL to Conjuctive Form where all the existential quantifiers ∃ are removed or eliminated from the predicate statements. It is done by using the existential quantifier with Skolem constant


   Steps for CNF conversion:

  • Step 1: Remove Implications ⇒, →, ⇔, ↔ It can be done by using: α → β ⩵ ¬ α ∨ β and α ⇔ β ⩵ (α → β) ∧ (β → α)
  • Step 2: Standardize variable It means use new variable for each new statements
  • Step 3: Move negation inwards It can be done as
    ¬ (∀(x) P(x)) = ∃ ¬ p(x)
    ¬ (∃(x) P(x)) = ∀ ¬ p(x)
    ¬ (α ∨ β) = ¬ α ∧ ¬ β
    ¬ (α ∧ β) = ¬ α ∨ ¬ β
    ¬ (¬ α) = α
  • Step 4: Skolemnization It is the process of removing the existential quantifier from the statements and replacing them with a constant called Skolem constant
  • Step 5: Drop univeral quantifier Remove the universal quantifier from the statements.


   "Anything anyone eats and is not killed by is Food" to FOL:
   ∀x ∀y eats(y, x) ^ ¬ killed_by(y) → food(x)

   Conversion to CNF:
   1: ¬(∀x ∀y eats(y, x) ^ ¬ killed_by(y)) ∨ food(x)
   2: same, as all of is single statement
   3: ¬ ∀x ∀y eats(y, x) ∨ killed_by(y) ∨ food(x)
   4: same, as no existential quantifier (∃)
   5: ¬ eats(y, x) ∨ killed_by(y) ∨ food(x)

Briefly distinguish between Antecedent mode and Consequent mode. What do you understand by Constraint Satisfaction? Mention its steps and an example.

->

Antecedent mode Consequent mode
It is the reasoning system where we start from facts and move towards goal. It is the reasoning system where we start from conclusion and move towards the fact.
Also called event-driven, forward reasoning Also called goal-driven, backward reasoning
Used for determining dieases from given symptoms. Used for determining symptoms from given disease.


   Constraint Satisfaction is the process of finding solution to a problem that imposes a pre condition or constraint which must be satisfied.
   It consists of the following steps:
  • Constraint are discovered and propagated as far as possible throughout the system.
  • If no solution, search begins. A guess is made and added as new constraint which can be propagated and continue to finding solution.


   For example: SUDUKO: Here we are given a pre condition of each row and column containing 1 to 9 without repetition and also each 3*3 section of the suduko must have 1 - 9 without repetition.

   We started by discovering those conditions and propageted to each cell of the suduko board. Then we start putting numbers in each cell considering the constraints imposed and find solution progressively.
