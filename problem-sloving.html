Problem Solving

Problem solving is one of the important task of any AI system and it is done in a sequence of actions.


For a agent to have a problem solving skill, it must be desinged as Formulate, Search and Execute
Here Formulate Refers to developing the alternative solutions to the problem, Search Refers to selecting the best alternative of all, and Execute refers to implementing the selected solution to solve the problem, if not solved repeat.


Well Defined Problem(WDP)

A well defined problem has the followinf 4 components:

  • Initial State

    It is the state that agent starts in, it can be any random possible scenario.

  • Possible State

    Description of possible actions available to agent

  • Goal test function

    It is a function that checks if the current state is the goal state or not

  • Path Cost Function

    It is a function that calculates the cost to each state from the pervious state or initail state


In each step the goal test function and path cost function is executed.
The performance of agent = summation of all cost of paths from intial state to goal state.


Planning Algorithm

It is the process of planning on how to proceed with each goal on the algorithm to be designed.

The two types of planning algorithms are:

  • Linear Planning
    • It is the planning algorithm type in which each goal is solved one after another in a sequential order.
    • It uses Goal Stacks to accomplish its planning.
    • It reduces search space as not all possible goal are considered.
  • Non-Linear Planning
    • It is the planning algorithm type in whihc multiple goal are solved at a time but not in parallel.
    • It uses Goal Set to accomplish its planning
    • It has larger search space since all possible goal ordering may have to be considered

Some properties of planning algorithm are:
  • Soundness: A planning algorithm is sound if all the predefiend conditions and goals are meet without violating any constraints.
  • Completness: A planning algorithm is complete if a solution is found whenever one actually exists
  • Optimality: A planning algorithm is optimal if the order in which solution are found is consistent with some measure of plan quality.


Production Rule system

It is a simple system that is build upon a production rule with condition and action.
A production rule consists of following components:

  • A set of rules: Each consisting of left side to hold the condition and right side for holding the action
  • One or more knowledge base: Information about the current problem
  • A control strategy: Specifying the order in which rule will be compared to database
  • A rule applier: To apply the rule
