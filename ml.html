Machine Learning

Machine Learning is a part of Artificial Intelligence that deals with the automatic learning process of the system.

It provides the ability to automatically learn and improve from experience without being explicitly programmed.
The ability to learn by itself after the initial dataset and improve upon the problems and further become accurate.


Components of Machine Learning

  • Learning Element: Responsible for making improvement
  • Performance Element: Responsible for selecting appropriate actions to environment
  • Critic: Responsible for telling how well the agent is doing
  • Problem Generator: Responsible for exploratory actions to the performance element

Methods of machine learning

  • Supervised Learning
    • In this method the dataset(input) and the correct output to those inputs are provided to the system, from which the system learns and further modify the algorithm to match the outputs more accuratetly in the future.
    • It is used in Regression and Classification
  • Unsupervised Learning
    • In this method only the dataset(input) are provided and no output is provided, from the given dataset the system has to find the pattern and predict the output of the problem.
    • It is used in Clustering systems
  • Reinforcement Learning
    • In this method there are no dataset(input) and output provided to the system, only the environment is provided, the system works in a cause and effect idea where it interacts with the environment using actions (cause) and analyiz the output of the actions (effect) which is either good(reward) or bad(failure).
    • It is used in the complex systems like Game playing.

Rote Learning (Parrot learning/Data Caching)

It is a simple learning method for Machine Learning systems where the system don't calculate but memorize the outpt. In other words this system simply learn the problem in repetition and memorize the output of certain input that is used when problem is put forward to it rather than recalculating.


For this system to work it needs:

  • Organized storage of information: Information must be organized and quickly accessible.
  • Generalization: To keep the large amount of information down to a manageable level.


Types of Learning

  • Learning in problem solving
    • It is the method of learning where it solves the problem and further improve upon it.
    • It may come up with qualitatively better solutions.
  • Learning by analogy
    • It is the method of learning where it observe or compare the problem with a significantly similar problem and try to solve it.
    • It is further divided into two type:
      • Transformational Analogy
        • It is the process of transforming the solution of old problem into solution of current problem.
        • It doesnot look at the how old problem was solved but looks at the final solution only.
      • Derivational Analogy
        • It is the process of deriving the solution of old problem into solution of current problem.
        • It looks at the how old problem was solved and try to derive the main structure of the solution to the new problem.
  • Learning by examples/Induction
    • It is the method of learning where the system uses examples to come up with an solution to the problem.
    • For example alphabet recognition system, where it has examples of alphabet and with the given alphabet it compares with its example and try to find the answer.
  • Explanation based learning
    • It is the method of learning where the system has analytical and knowledge intensive approach.
    • It is a generalization to learning by examples, where it uses only one example and try to explain why that example is used as training concept.
    • For example: the Fork trap is a concept of attacking in chess where the opponent has to lose its one piece inorder to save its another piece(usually king). The system can study one example of such move and try to explain why is that happening hence can use this move in real game later in its advantage.
    • For this system to work it needs the following Components:
      • A training example: What learning program sees in the world.
      • A goal Concept: A high level description of what the program has to learn.
      • An operational criterion: A description of which concepts are usable.
      • A domain theory: A set of rules that describe the relationship between objects and action in domain.

Winston's Learning Program

  • It is the early structural concept learning program developed by Patrick Henry Winston(MIT)
  • It introduces the concept of near miss which is an object that is not the instance of the concept but that is very similar to such instances.
  • The basic approach that winston's program took to the problem of concept formationcan be described as:
    • Beign with a structural description of one known instance of the concept. Call that description the concept description.
    • Examine description of other known instances of the concept. Generalize the definition to include them.
    • Examine description of near misses of the concept. Restrict the definition to exclude them.
